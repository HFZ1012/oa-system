# 项目核心技术实现参考手册 (登录、鉴权与缓存)

本手册总结了当前项目基于 `vue3-admin-better` 与 `Spring Security + Redis` 实现的核心架构逻辑，旨在为其他同类项目提供技术参考。

---

## 1. 登录与身份认证 (JWT + Redis 滑动窗口)

### 1.1 核心逻辑
- **双重控制**：使用 JWT 作为前端通行证，使用 Redis 作为后端状态控制器。
- **滑动过期**：JWT 本身设置长效过期，但 Redis 中的 Token Key 设置为 **2 小时** 有效期。每次合法 API 调用均会刷新 Redis 寿命。

### 1.2 后端实现关键点
- **Token 生成**：登录成功后生成 JWT 并存入 Redis。
- **过期刷新**：在 `TokenService` 中实现滑动逻辑。
```java
// TokenService.java
public void refreshTokenOnLegalCall(String token) {
    // 每次请求成功后，将 Redis 中的有效期重置为 2 小时
    redisTemplate.opsForValue().set(TOKEN_KEY + token, userDetails, 2, TimeUnit.HOURS);
}
```

## 2. 鉴权与性能优化 (缓存优先策略)

### 2.1 鉴权流程
1. **拦截请求**：`JwtAuthorizationFilter` 拦截所有 `/api/**` 请求。
2. **读缓存**：优先从 Redis 中通过 Token 获取 `UserDetails`（包含角色和权限）。
3. **读库备份**：若缓存失效，则调用 `UserDetailsServiceImpl` 查询数据库，并重新填充 Redis。

### 2.2 数据库投影查询
为了减轻 Oracle 压力，角色和权限的读取应使用 JPA 投影查询，避免加载完整的 User 实体。
```java
// SysUserRepository.java
@Query("select r.roleCode from SysRole r join r.users u where u.id = ?1")
List<String> findRoleCodesByUserId(Long userId);
```

---

## 3. 角色与权限控制 (RBAC)

### 3.1 前端体现
- **页面级控制**：在路由配置文件中通过 `meta.roles` 定义准入角色。
- **按钮级控制**：使用 `v-permissions` 指令。后端返回的 `permissionCodes` 会在前端 `store` 中保存，用于动态移除无权 DOM。

### 3.2 权限表结构
采用标准 5 表：`sys_user`, `sys_role`, `sys_permission`, `sys_user_role`, `sys_role_permission`。

---

## 4. 登出逻辑 (状态闭环)

### 4.1 彻底清理逻辑
登出不仅仅是前端清除 Token，必须包含以下三个步骤：
1. **通知后端**：调用 `/api/auth/logout`，后端执行 `redisTemplate.delete(TOKEN_KEY + token)`。
2. **内存清理**：重置 Vuex/Pinia 中的用户信息。
3. **状态重置**：**关键点**，使用 `window.location.href = "/#/login"` 进行硬跳转，而非 `router.push`。这能彻底重置 Vue Router 的路由栈，防止动态路由递归报错。

## 5. 关键 Bug 修复经验

### 5.1 请求拦截器数据损坏
**现象**：使用 `lodash.pickBy` 过滤空参数时，如果 `config.data` 是字符串（如 Token），会被拆解成索引对象。
**修复**：增加类型判断。
```javascript
// request.js
if (config.data && typeof config.data === 'object') {
  config.data = pickBy(config.data, identity);
}
```

### 5.2 登出后再次登录递归报错
**现象**：`Maximum call stack size exceeded`。
**原因**：`resetRouter()` 配合 `router.push` 在特定守卫下触发无限重定向。
**解决**：改用硬跳转刷新页面。

## 6. 环境建议
- **后端**：Spring Boot + Spring Security + Spring Data JPA。
- **缓存**：Redis (必须，用于 Token 寿命管理)。
- **前端**：Vue 3 + Vite/Rspack + vue3-admin-better。
