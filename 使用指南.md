# Vue3 Admin Better 日常使用指南

这份指南是为了帮助您在本地部署成功后，能够快速上手日常开发和维护本项目（基于 Vue 3.0 + Element Plus + Rspack + Express + 达梦数据库）。

## 1. 环境依赖与准备

本项目依赖于以下基础环境：
- **Node.js**: 建议版本 >= 16.0.0 (开发环境建议使用 v18+)
- **包管理器**: `pnpm` (建议版本 >= 10.x)
- **数据库**: **达梦数据库 (DM8)**，安装在 `192.168.0.222:5236`

如果在其他电脑上重新部署，只需执行：
```bash
#1. 克隆代码后进入项目根目录
cd vue3

#2. 安装前端依赖
pnpm install

#3. 安装后端依赖
cd backend
pnpm install
cd ..
```

## 2. 常用开发命令

在日常开发过程中，您需要启动后端接口服务器以及前端开发服务器：

- **第一步：启动后端真实 API 服务器**：
  ```bash
  cd backend
  node server.js
  ```
  后端服务将运行在 `http://localhost:3000`。
  *首次启动会自动连接达梦数据库并初始化 `MOCK_TABLE`、`TASK_TABLE` 和 `MEETING_APPLICATION` 表。*

- **第二步：启动前端开发服务器** (支持热重载，另开一个终端执行)：
  ```bash
  pnpm run serve:rspack
  ```
  启动后，在浏览器访问 `http://localhost:8092/` 即可预览项目。

## 3. 项目核心目录结构解析

了解目录结构有助于您快速定位需要修改的代码：

- 📂 `src/` - **前端源代码目录**
  - 📁 `api/`：存放所有前端的接口请求函数（已部分对接真实后端，如 `table.js`）。
  - 📁 `views/`：存放所有的 Vue 页面组件（如工作台、表格、表单页面等）。
  - 📁 `router/`、`store/`、`components/`、`styles/` 等：前端核心配置与资源。
- 📂 `backend/` - **真实后端 API 目录** (基于 Express + dmdb 驱动)
  - 📄 `server.js`：后端入口文件，包含达梦数据库连接（dmdb）和具体的增删改查接口。
- 📄 `init.sql` - **达梦数据库初始化脚本备份** (供手动参考，server.js 具备自动建表功能)。
- 📂 `mock/` - **本地 Mock 数据目录** (对于尚未对接真实后端的接口，仍可在此处使用假数据)。
- 📄 `rspack.config.js` - **Rspack 构建配置文件**。

## 4. 日常全栈开发工作流示例

### 4.1 数据库连接信息

### 4.2 编写后端 API 接口
在 `backend/server.js` 中新增 Express 路由。注意使用达梦/Oracle 兼容语法：
```javascript
app.post('/api/myData', async (req, res) => {
  try {
    const result = await execute('SELECT * FROM my_new_table');
    res.json({ code: 200, msg: 'success', data: result.rows });
  } catch (error) {
    res.status(500).json({ code: 500, msg: 'Error' });
  }
});
```

### 4.3 前端对接 API
在 `src/api/` 目录下新建请求函数，指向真实的后端地址：
```javascript
import request from '@/utils/request'

const REAL_API_BASE = 'http://localhost:3000'

export function getMyData(data) {
  return request({ 
    url: `${REAL_API_BASE}/api/myData`, 
    method: 'post',
    data
  })
}
```

## 5. 常见问题与注意事项

1. **前端端口变更**：
   前端开发端口已从 `8091` 迁移至 `8092`。
2. **达梦数据库报错**：
   - 报错 `[6071] 消息加密失败`：请确保连接字符串中包含 `loginEncrypt=false`。
   - 语法错误：达梦高度兼容 Oracle，若遇到关键字冲突（如 `percent`），请加双引号 `\"percent\"`。
3. **Mock 与真实 API 共存**：
   部分接口（如表格、任务、会议室）已指向真实后端，而登录等接口仍走 Mock 逻辑。
