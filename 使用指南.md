# Vue3 Admin Better 日常使用指南

这份指南是为了帮助您在本地部署成功后，能够快速上手日常开发和维护本项目（基于 Vue 3.0 + Element Plus + Rspack + Express + MySQL）。

## 1. 环境依赖与准备

本项目依赖于以下基础环境：
- **Node.js**: 建议版本 >= 16.0.0
- **包管理器**: `pnpm` (当前使用的是 10.x 版本)
- **Docker & Docker Compose**: 用于快速启动 MySQL 数据库环境

如果在其他电脑上重新部署，只需执行：
```bash
#1. 克隆代码后进入项目根目录
cd vue3

#2. 安装前端依赖
pnpm install

#3. 安装后端依赖
cd backend
npm install
cd ..
```

## 2. 常用开发命令

在日常开发过程中，您需要启动数据库、后端接口服务器以及前端开发服务器：

- **第一步：启动 MySQL 数据库** (首次启动会自动执行 `init.sql` 建表并插入数据)：
  ```bash
  docker-compose up -d
  ```

- **第二步：启动后端真实 API 服务器**：
  ```bash
  cd backend
  node server.js
  ```
  后端服务将运行在 `http://localhost:3000`。

- **第三步：启动前端开发服务器** (支持热重载，另开一个终端执行)：
  ```bash
  pnpm run serve:rspack
  ```
  启动后，在浏览器访问 `http://localhost:8091/` 即可预览项目。

## 3. 项目核心目录结构解析

了解目录结构有助于您快速定位需要修改的代码：

- 📂 `src/` - **前端源代码目录**
  - 📁 `api/`：存放所有前端的接口请求函数（已部分对接真实后端，如 `table.js`）。
  - 📁 `views/`：存放所有的 Vue 页面组件（如工作台、表格、表单页面等）。
  - 📁 `router/`、`store/`、`components/`、`styles/` 等：前端核心配置与资源。
- 📂 `backend/` - **真实后端 API 目录** (基于 Express + MySQL)
  - 📄 `server.js`：后端入口文件，包含数据库连接和具体的增删改查接口（如 `/table/getList`）。
- 📄 `docker-compose.yml` & `init.sql` - **数据库容器化配置与初始化脚本**。
- 📂 `mock/` - **本地 Mock 数据目录** (对于尚未对接真实后端的接口，仍可在此处使用假数据)。
- 📄 `rspack.config.js` - **Rspack 构建配置文件**（替代传统的 Webpack）。

## 4. 日常全栈开发工作流示例

我们以新增一个页面和数据表为例：

### 4.1 数据库操作
如果需要新增表或字段，可以使用数据库管理工具（如 Navicat, DataGrip）连接本地的数据库：
- **Host**: `127.0.0.1` (或 `localhost`)
- **Port**: `3306`
- **Username**: `root`
- **Password**: `root`
- **Database**: `vue3_admin`

### 4.2 编写后端 API 接口
在 `backend/server.js` 中新增 Express 路由。例如：
```javascript
app.post('/api/myData', async (req, res) => {
  try {
    const [rows] = await pool.query('SELECT * FROM my_new_table');
    res.json({ code: 200, msg: 'success', data: rows });
  } catch (error) {
    res.status(500).json({ code: 500, msg: 'Error' });
  }
});
```
*提示：修改 `server.js` 后，需重启 node 进程。如果希望自动重启，可以安装 `nodemon`。*

### 4.3 前端对接 API
在 `src/api/` 目录下新建请求函数，指向真实的后端地址：
```javascript
import request from '@/utils/request'

const REAL_API_BASE = 'http://localhost:3000'

export function getMyData(data) {
  return request({ 
    url: `${REAL_API_BASE}/api/myData`, 
    method: 'post',
    data
  })
}
```
然后在您的 Vue 页面组件 (`src/views/`) 中引入该 API 即可渲染真实数据。

## 5. 常见问题与注意事项

1. **Mock 与真实 API 共存**：
   本项目目前采用“混合模式”过渡。部分接口（如表格的增删改查）已指向真实后端的 `http://localhost:3000`，而如登录（`/login`）等尚未改造的接口仍走本地 Mock 逻辑。您可以按需逐步将需要的接口迁移到 `backend/server.js`。
2. **数据库连接失败**：
   请确保 Docker Desktop 已启动，并且 `docker-compose up -d` 执行成功。如果 3306 端口被本地其他 MySQL 占用，可以在 `docker-compose.yml` 中修改映射端口（如 `3307:3306`）。
3. **前端打包与部署**：
   当开发完成后，可执行 `pnpm run build` 进行前端生产环境打包。打出的 `dist` 目录配合后端的 `server.js` 即可部署到生产服务器。
