# 上海分院内网系统使用指南

本指南用于帮助项目成员在本地完成安装后，快速开展前端开发、后端联调和日常维护工作。

当前项目的实际技术栈如下：

- 前端：`Vue 3` + `Element Plus` + `Rspack`
- 后端：`Express`
- 数据库：`达梦 DM8`
- 接口模式：`Mock` 与真实后端并存

## 1. 环境准备

建议使用以下环境：

- `Node.js >= 16`，推荐 `v18+`
- `pnpm >= 10`
- 可访问的达梦数据库：`192.168.0.222:5236`

首次在新电脑拉起项目时执行：

```bash
# 进入项目根目录
cd SHBCAS

# 安装前端依赖
pnpm install

# 安装后端依赖
cd backend
pnpm install
cd ..
```

## 2. 常用命令

日常开发通常需要同时启动后端和前端。

### 2.1 启动后端

```bash
cd SHBCAS/backend
node server.js
```

说明：

- 后端地址：`http://localhost:3000`
- 首次启动会自动初始化以下表：
  - `mock_table`
  - `task_table`
  - `meeting_application`
- 数据库连接写在 `backend/server.js` 中，当前连接串包含 `loginEncrypt=false`

### 2.2 启动前端

在项目根目录另开一个终端执行：

```bash
cd SHBCAS
pnpm run serve:rspack
```

说明：

- 前端开发地址：`http://localhost:8092/`
- 当前项目标题配置为：`上海分院内网系统`
- 开发端口来自 `src/config/setting.config.js` 中的 `devPort`

### 2.3 打包前端

```bash
cd SHBCAS
pnpm run build
```

如需构建并打包压缩文件：

```bash
cd SHBCAS
pnpm run build:zip
```

## 3. 项目结构速览

以下目录是日常开发最常接触的部分：

- `src/`
  - `views/`：页面组件目录，包含会议、任务、车辆、物资、用印、差旅等业务页面
  - `api/`：前端接口封装
  - `config/`：标题、端口、网络配置等
  - `router/`、`store/`、`utils/`：路由、状态管理与工具方法
- `backend/`
  - `server.js`：真实后端入口，包含数据库连接、建表逻辑和接口实现
  - `package.json`：后端依赖声明
- `mock/`
  - 本地 Mock 数据与 Mock 路由
  - 对于尚未接入真实后端的功能，仍由这里提供数据
- `rspack.config.js`
  - 前端构建与开发服务器配置
- `init.sql`
  - 达梦初始化脚本备份，可作为手工排查参考

## 4. 当前接口对接现状

目前项目不是“全量真实后端”，而是“部分真实接口 + 部分 Mock”。

### 4.1 已接真实后端的模块

以下前端接口文件已直接请求 `http://localhost:3000`：

- `src/api/table.js`
- `src/api/task.js`
- `src/api/meeting.js`

对应后端接口位于 `backend/server.js`，当前包含：

- 表格模块：`/table/getList`、`/table/doEdit`、`/table/doDelete`
- 任务模块：`/task/getList`、`/task/doAdd`、`/task/doEdit`、`/task/doDelete`
- 会议模块：`/meeting/getList`、`/meeting/doAdd`、`/meeting/updateStatus`、`/meeting/doDelete`

### 4.2 仍走 Mock 的部分

未单独改造成真实后端的接口，仍会走前端默认配置中的 Mock 地址：

- 默认 `baseURL` 配置在 `src/config/net.config.js`
- 开发环境默认请求地址为：`/vab-mock-server`

这意味着：

- 登录、权限、部分演示页面等功能，通常仍依赖 Mock
- 若新增业务模块未显式写死真实后端地址，则默认不会自动走 `localhost:3000`

## 5. 日常开发方式

### 5.1 新增真实后端接口

在 `backend/server.js` 中新增 Express 路由，推荐沿用现有返回格式：

```javascript
app.post('/example/getList', async (req, res) => {
  try {
    const result = await execute('SELECT * FROM example_table');
    res.json({
      code: 200,
      msg: 'success',
      data: result.rows,
    });
  } catch (error) {
    res.status(500).json({
      code: 500,
      msg: 'Error',
    });
  }
});
```

建议保持以下约定：

- 成功返回 `code: 200`
- 失败返回 `code: 500`
- 查询结果统一经过必要格式处理后再返回前端
- SQL 尽量使用达梦兼容语法

### 5.2 前端对接真实接口

若某个模块要改成真实后端，最直接的方式是在 `src/api/` 中单独封装：

```javascript
import request from '@/utils/request'

const REAL_API_BASE = 'http://localhost:3000'

export function getExampleList(data) {
  return request({
    url: `${REAL_API_BASE}/example/getList`,
    method: 'post',
    data,
  })
}
```

说明：

- 当前项目中，表格和会议模块使用 `post`
- 任务列表接口 `getList` 当前使用 `get`
- 新增接口时，前后端请求方式要保持一致

### 5.3 修改页面时的定位方式

可以按下面的习惯定位代码：

- 改页面展示：优先找 `src/views/`
- 改接口请求：找 `src/api/`
- 改开发端口、标题、登录拦截等通用配置：找 `src/config/`
- 改真实数据库逻辑：找 `backend/server.js`

## 7. 注意事项

- 前端开发端口当前是 `8092`，不是 `8091`
- 达梦若报 `[6071] 消息加密失败`，请检查连接串是否包含 `loginEncrypt=false`
- 达梦对关键字较敏感，如字段名为 `percent` 时，需要写成 `"percent"`
- 后端没有单独的启动脚本，当前就是直接执行 `node server.js`
- 前端虽然全局默认走 Mock，但 `table`、`task`、`meeting` 三个模块已绕过默认配置，直接请求真实后端
